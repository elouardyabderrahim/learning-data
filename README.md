# DATA:
### Data catalog for data lakes:
- what is the source of this data?
- where is this data used?
- who is the owner of this data?
- how often is this data updated?
- good practice in terms of data governance?
- Ensre reproducibility
- No catalog --> data swamp :  a data lake becomes a "data swamp," a disorganized collection of raw, uncurated, and possibly unusable data.
- good practice for any data storage solution
  - Reliability
  - Autonomy
  - Scalability
  - Speed
